# 部署说明：把页面发布成公开网址（免费 · 免服务器 · 免备案）

这套方案用 GitHub 免费自动化，每 4 小时自动抓取一次数据、生成网页，
别人用一个网址就能访问。你只需按下面做一次，之后全自动，不用管。

最终网址形如：`https://你的用户名.github.io/仓库名/`

---

## 一、准备（约 10 分钟，只做一次）

**1. 注册 GitHub 账号**
打开 <https://github.com> → Sign up，注册一个账号（免费）。

**2. 新建一个仓库（Repository）**
登录后点右上角 “+” → “New repository”。
- Repository name：随便起，例如 `luan-water`
- 选择 **Public（公开）**（GitHub 免费版发布网页要求仓库公开；本项目没有任何密码或私密信息，公开无妨）
- 其他默认，点 “Create repository”。

**3. 上传本文件夹里的所有文件**
在新仓库页面点 “Add file” → “Upload files”，
把本文件夹里的**全部内容**（包括 `app.py`、`generate.py`、`template.html`、
`requirements.txt`、`docs` 文件夹、`.github` 文件夹）一起拖进去。
> 直接把整个文件夹的内容框选后拖入上传区，GitHub 会自动保留子文件夹结构。
> 如果 `.github/workflows/build.yml` 没传上去，见文末“补传工作流文件”。

拖完后，页面下方点 “Commit changes”。

---

## 二、开启网页发布（Pages）

1. 进入仓库的 **Settings（设置）** → 左侧 **Pages**。
2. 在 “Build and deployment” 下：
   - Source 选 **Deploy from a branch**
   - Branch 选 **main**，文件夹选 **/docs**，点 **Save**。
3. 等 1~2 分钟，刷新此页，顶部会显示你的网址
   `https://你的用户名.github.io/仓库名/`。打开就能看到页面。

到这里就已经能用了——`docs/index.html` 里已含一份最新数据。

---

## 三、让它自动更新（Actions）

数据的自动刷新由 GitHub Actions 完成，已经配置好，默认**每 4 小时**跑一次。

- 想**立刻**更新一次：进入仓库 **Actions** 标签 → 左侧选
  “构建六安水利招采页面” → 右侧 **Run workflow** → 绿色按钮。
  跑完后网页数据即刷新。
- 第一次进 Actions 若提示是否启用工作流，点 **同意/Enable** 即可。

之后你什么都不用做，它会自动保持更新。

---

## 四、常见问题

**改更新频率？**
编辑 `.github/workflows/build.yml` 里的 `cron: "0 */4 * * *"`。
`*/4` 表示每 4 小时；改成 `*/2` 就是每 2 小时，`*/8` 就是每 8 小时。

**改“水利”关键词范围？**
编辑 `app.py` 顶部的 `KEYWORDS` 列表，增删词后提交即可，下次构建生效。

**某次自动抓取失败会怎样？**
交易中心偶有临时防护。抓取失败时程序会**保留上一次的好页面**，
网站不会变空白，下次计划任务自动重试。

**国内访问速度？**
`github.io` 在国内一般能打开，偶尔偏慢。几十人小范围够用。
若以后想更稳更快（加国内 CDN 或迁到国内托管），告诉我，我再帮你弄。

**定时任务会不会自己停？**
GitHub 规定：仓库长期无提交会暂停定时任务。本方案每次自动构建都会
提交一次（页面时间戳每次都变），因此会持续保持活跃，通常不用担心。
万一发现长期不更新，去 Actions 点一次 “Run workflow” 即可重新唤醒。

**页面已注明数据来源**并保留了每条公告的原文链接，属规范的二次展示。

## 五、补传工作流文件（仅当 `.github` 没传上去时）

有些浏览器拖拽不会上传以 `.` 开头的隐藏文件夹。若 Actions 标签里看不到
工作流，按下面手动补：
1. 仓库首页点 “Add file” → “Create new file”。
2. 文件名处输入：`.github/workflows/build.yml`（斜杠会自动分层）。
3. 把本文件夹内 `.github/workflows/build.yml` 的内容整段粘贴进去。
4. 点 “Commit changes”。

---

## 备选：如果 GitHub 的服务器抓不到数据

GitHub 的构建服务器在境外，极个别情况下政府站可能对境外访问有限制，
导致自动抓取持续失败（Actions 里报抓取错误、网站长期不更新）。
若遇到这种情况，可改成“在国内一台常开的电脑上定时运行 `generate.py`
再推送到 GitHub”。需要时告诉我，我给你现成的一键脚本和设置步骤。
